### 1、github的仓库怎么创建？

* 仓库设为 public 即可
* 周记包含你的`学习笔记.md（本周你学到的内容）`、`生活随记.md（如果有）`
* 可以给飞书知识库的文档**在线链接**，在`学习笔记.md`中把链接填进去即可

```markdown
根目录，点开你仓库网址后看到的那一页
├── 📁 week01
│   ├── 📁 project         # 第一周项目代码
│   └── 📁 journal         # 第一周周记（Markdown格式）
└── 📁 week02-03
    ├── 📁 project         # 第二、三周合并后的项目代码
    └── 📁 journal         # 第二、三周学习记录 / AI 使用说明等
```

### 2、考核的形式是什么，有第二轮考核吗？

* 第一轮考核的题目仍然是同一个“宿舍报修管理系统”项目
  * 目前文档发布方式为：**第一周单独发布，第二周与第三周任务合并发布**
    * 第一周：后端开发，数据库交互，DAO(*Data Access Object*)层、service 层开发
    * 第二、三周合并：Spring Boot 改造、RESTful 接口、鉴权、前端开发与前后端联调
  * 基础好的同学可以直接往后做

* 第二轮考核有两周（一轮考核结束后开始，**4月19日晚上23点59分**截至）
  * 有三道题目供大家选择，选择自己感兴趣的那道题目就行，**只需要完成其中一道**
  * **4月19日后**，需要准备项目答辩（面向指导老师），具体时间后期会通知

### 3、分组是成员之间要互相合作吗？为啥分组

* 不是，每个人独立完成项目，本次招新考核均为个人考核，不涉及合作开发
* 分组是分配一个师兄师姐作为导师，便于管理，由于每个人进度不一，有不好意思问 / 对考核疑惑的问题可以咨询导师
  * 导师可以为你提供思路讲解，但是你不能寄希望于导师帮你把代码都写完
